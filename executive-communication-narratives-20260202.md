# 向高層溝通的敘事框架（2026 更新）

Staff 工程師向高層溝通的目標不是「說服」，而是讓高層能更快做決策。重點是「情境 → 風險 → 選項 → 建議 → 需要的決策」。

## 一頁敘事結構（建議順序）

1. 問題與影響範圍（為什麼現在重要）
2. 目前風險與代價（不做會發生什麼）
3. 可選方案與權衡（成本、時間、風險）
4. 推薦路徑與關鍵假設
5. 需要的決策與下一步

## 高層常問的問題

1. 這件事如果不做，最壞會怎樣？
2. 需要多少資源？會排擠什麼？
3. 成功的定義是什麼？怎麼知道「夠好」？
4. 哪些假設最脆弱？如何及早驗證？

## Staff 工程師的實作要點

1. 讓風險可視化（時間、成本、可信度）
2. 使用「最小可決策單位」切分議題
3. 以可反駁的假設呈現建議
4. 把「不確定」寫出來，避免會後反轉
